## 优化方案

1. 明确“可用版本”仅指稳定版，并形成可执行的展示规则说明。
   验证：用至少 3 组历史稳定版样例人工对照规则，确认展示结果唯一且无重复或缺失。
2. 确定版本列表默认按发布日期降序排序。
   验证：使用 3 组不同发布日期样例人工检查，确认排序始终为最新在前。
3. 固定版本条目最小必需信息为：版本号、发布日期、适用平台、更新说明摘要。
   验证：随机抽查 3 条记录，确认四项字段均存在且可读。
4. 明确“关于页面”仅呈现版本号、作者、官网、版权、更新渠道。
   验证：逐项人工核对页面字段，确保只包含以上 5 项且无多余信息。
5. 回归检查版本列表与关于页面在常见分辨率下的显示效果。
   验证：在常见分辨率下人工检查，无文本截断、无重叠。
